Pasta Filtros

É na pasta Filtros que ficarão, bem, os seus filtros. São as classes que serão referênciadas no seu arquivo de configuração de filtros, sendo uma para cada filtro.

Estes filtros são como os conhecidos "middleware": uma camada de código que é executada antes que a requisição seja de fato processada. Desse modo, você têm a oportunidade de fazer validações ou o que desejar antes de o usuáio chegar ao seu recurso.

Criando um Filtro

Para criar um novo filtro basta utilizar a forja: 

php forja criar filtro exemploFiltro

Um novo arquivo com o nome desejado será criado em seu diretório de filtros: 

<?php
                                    
namespace App\Filtros;

class ExemploFiltro
{
    /**
    * Aplica o filtro configurado
    */
    public function aplicar()
    {
        // lógica do seu filtro, normalmente retornará um redirecionar
    }
}

Utilização

Filtros devem sempre conter o método aplicar, que é utilizado pelo framework para de fato executar o filtro desejado antes da resposta da requisicao.

Por exemplo, você pode criar um filtro para impedir usuários que já estão logados de acessarem novamente a página de login e redireciona-los para sua página principal: 

public function aplicar()
{
    if (sessao('usuario')) {
        return redirecionar('/home');
    }
}

Ou um outro de rotas exclusivas para usuários logados, que caso contrário irá redirecionar para página de login, talvez com algum tipo de informação sobre o redirecionamento: 

public function aplicar()
{
    if (! sessao('usuario')) {
        return redirecionar('/home')->com('mensagem', 'Você precisa estar logado para acessar esta página');
    }
}
