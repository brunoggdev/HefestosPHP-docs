Pasta Models

Na pasta Models ficarão, claro, os seus Models, ou seja, a camada do seu projeto responsável por acessar o seu banco de dados.

Cada Model está diretamente associada à uma de suas tabelas no banco de dados e irá interagir com ela para realizar as operações desejadas.

Antes de mais nada, visto que as models trabalham diretamente com ela, é imprescindível conferir a classe Database, esta que se conecta ao banco de dados e provê métodos que podem ser encadeados para interagir com ele.

Desse modo, suas models herdam métodos auxiliares para facilitar ainda mais essa interação com o banco ao configurar automaticamente suas operações para a tabela especificada.

Busncando dados

O método select() inicia a consulta no banco, podendo buscar colunas específicas informando um array opcional ou todas por padrão. Este método irá retornar a instância da Database onde você poderá usar o método todos() para retornar todos os resultados ou primeiro() para retornar, claro, apenas o primeiro resultado.
