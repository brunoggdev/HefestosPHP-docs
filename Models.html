Pasta Models

Na pasta Models ficarão, claro, os seus Models, ou seja, a camada do seu projeto responsável por acessar o seu banco de dados.

Cada Model está diretamente associada à uma de suas tabelas no banco de dados e irá interagir com ela para realizar operações de CRUD (Create, Read, Update, Delete).

Para tal, cada model se conecta automaticamente com o banco de dados e provê alguns metodos auxiliares para tornar a interação com o banco ainda mais simples.

  • Criando um Model
  • Recuperando dados
  • Inserindo dados
  • Atualizando dados
  • Deletando dados
