Pasta Testes

Neste diretório você pode criar testes para as funcioalidades do seu projeto, uma parte essencial para garantir o devido funcionamento do software com o passar do tempo e as modificações nele feitas.

Para tanto, o HefestosPHP conta com um sistema de testes nativo, inspirado pelo incrível framework de testes Pest, focado em simplicidade e legibilidade dos testes.

Você pode agrupar seus testes relacionados em arquivos e estes arquivos em pastas. Depois, executar todos os testes de uma vez, pastas especificas ou arquivos específicos.

  • Escrevendo testes
  • Executando testes
  • Aprofundando

Escrevendo testes

Para escrever testes primeiro crie um arquivo neste diretório com o nome que desejar. Aqui você terá acesso à mais quatro funções específicas de testes: testar(), esperar(), usar() e falhar().

A função testar() adiciona seus testes na suíte de testes, recebendo como primeiro parâmetro uma descrição para o teste que será visualizada no terminal e, como segundo parâmetro, uma função com o teste de fato.

Nesta função você pode executar quaisquer rotinas que deseje testar, como conferir models, arquivos, APIs, etc. O teste passa a menos que uma espera (função esperar()) falhe, a função falhar() seja usada, ou, claro, caso ocorra um erro na execução. Por exemplo:

Executando testes

Uma vez declarados os testes, você deve utilizar a forja para executa-los e os resultados serão exibidos no seu terminal:

php forja testar

Como dito, você pode criar vários arquivos de testes e até agrupa-los em subpastas dentro da pasta Testes. Informe então o caminho da subpasta com os testes desejados ou mesmo um arquivo específico caso não deseje executar todos os testes:

php forja testar subpasta

php forja testar subpasta/testes_especificos.php

Aprofundando

A função esperar() é o coração dos testes pois promove uma api intuitiva para ser utilizada dentro dos testes e torna-los mais organizados e legiveis.

Ela recebe um único parâmetro que pode ser qualquer coisa (string, array, objeto, etc.) e retorna um objeto onde você pode encadear métodos para testar esse valor, de modo que você pode praticamente ler o teste e saber o que ele faz rapidamente.

Por fim, a função usar() pode ser usada para executar uma função informada e definir valores que podem ser compartilados por todos os testes que vierem depois. Para isso, basta que a função informada retorne um array associativo e então serão definidas proriedades de classe na suíte de testes (acessadas com $this) compartilhadas em todos os testes.
